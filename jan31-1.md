Database Tools\n\n```pseudo\nTOOL: query_database\n  INPUT: sql_query (string), limit (int, default=100)\n  \n  VALIDATE sql_query is SELECT only (no mutations)\n  CONNECT to PostgreSQL using DATABASE_URL\n  EXECUTE query with LIMIT applied\n  RETURN results as JSON array\n  \n---\n\nTOOL: list_tables\n  \n  QUERY pg_catalog.pg_tables WHERE schemaname = 'public'\n  RETURN table names with row counts\n  \n---\n\nTOOL: describe_table\n  INPUT: table_name (string)\n  \n  QUERY information_schema.columns WHERE table_name = input\n  QUERY pg_indexes for index information\n  QUERY pg_constraint for foreign keys\n  RETURN {columns, indexes, foreign_keys, sample_data}\n  \n---\n\nTOOL: analyze_query\n  INPUT: sql_query (string)\n  \n  RUN EXPLAIN ANALYZE on query\n  PARSE execution plan\n  RETURN {plan, estimated_cost, suggestions}\n```\n\n### 2. Rails Introspection Tools\n\n```pseudo\nTOOL: list_models\n  \n  GLOB app/models/**/*.rb\n  PARSE each file for class definitions\n  EXTRACT associations, validations, scopes\n  RETURN model metadata\n  \n---\n\nTOOL: list_routes\n  \n  EXECUTE `rails routes --expanded`\n  PARSE output into structured format\n  RETURN [{verb, path, controller, action}]\n  \n---\n\nTOOL: show_schema\n  INPUT: model_name (optional)\n  \n  READ db/schema.rb\n  IF model_name provided:\n    FILTER to relevant table\n  RETURN schema definition\n  \n---\n\nTOOL: list_migrations\n  INPUT: status_filter (all|pending|applied)\n  \n  EXECUTE `rails db:migrate:status`\n  PARSE migration list\n  READ migration files for descriptions\n  RETURN [{version, status, name, file_path}]\n```\n\n### 3. React Component Tools\n\n```pseudo\nTOOL: list_components\n  INPUT: path (optional, default=\"app/javascript/components\")\n  \n  GLOB #{path}/**/*.{jsx,tsx,js,ts}\n  FOR each file:\n    PARSE with babel\n    EXTRACT component name, props interface, hooks used\n    DETECT if uses Bootstrap classes\n  RETURN [{name, path, props, hooks, has_bootstrap}]\n  \n---\n\nTOOL: analyze_component\n  INPUT: component_path (string)\n  \n  READ file content\n  PARSE JSX/TSX with babel\n  EXTRACT:\n    - Component name and type (functional/class)\n    - Props with types\n    - State variables (useState)\n    - Effects and dependencies (useEffect)\n    - Custom hooks used\n    - Child components rendered\n    - CSS classes used (className attributes)\n    - Bootstrap classes present\n  RETURN full component analysis\n  \n---\n\nTOOL: find_component_usage\n  INPUT: component_name (string)\n  \n  GREP for import statements matching component\n  GREP for JSX usage <ComponentName\n  RETURN [{file, line, context}]\n  \n---\n\nTOOL: list_hooks\n  INPUT: custom_only (boolean, default=true)\n  \n  GLOB app/javascript/hooks/**/*.{js,ts}\n  PARSE each hook file\n  EXTRACT hook name, parameters, return value\n  RETURN hook definitions\n  \n---\n\nTOOL: analyze_props_flow\n  INPUT: component_path (string)\n  \n  BUILD component tree starting from target\n  TRACE prop drilling paths\n  IDENTIFY context usage\n  RETURN {prop_sources, drilling_depth, context_dependencies}\n```\n\n### 6. Bootstrap Analysis Tools\n\n```pseudo\nTOOL: scan_bootstrap_usage\n  INPUT: path (optional)\n  \n  DEFINE bootstrap_patterns = [\n    # Layout\n    \"container\", \"container-fluid\", \"row\", \"col-*\",\n    # Components  \n    \"btn\", \"btn-*\", \"card\", \"card-*\", \"modal\", \"navbar\",\n    \"nav\", \"nav-*\", \"dropdown\", \"badge\", \"alert\", \"alert-*\",\n    # Utilities\n    \"d-*\", \"flex-*\", \"justify-*\", \"align-*\",\n    \"m-*\", \"p-*\", \"mt-*\", \"mb-*\", \"mx-*\", \"my-*\",\n    \"pt-*\", \"pb-*\", \"px-*\", \"py-*\",\n    \"text-*\", \"bg-*\", \"border-*\",\n    \"w-*\", \"h-*\", \"rounded-*\", \"shadow-*\"\n  ]\n  \n  GLOB all JSX/TSX/ERB/HTML files\n  FOR each file:\n    EXTRACT all className values\n    MATCH against bootstrap_patterns\n    RECORD {file, line, classes, component_context}\n  \n  GROUP by Bootstrap category\n  RETURN {\n    summary: {total_files, total_usages, by_category},\n    details: [{file, usages}]\n  }\n  \n---\n\nTOOL: analyze_bootstrap_component\n  INPUT: component_type (string)  # e.g., \"modal\", \"card\", \"navbar\"\n  \n  SEARCH for Bootstrap component patterns\n  IDENTIFY:\n    - Files using this component type\n    - Variant classes used (btn-primary, card-header, etc.)\n    - Custom overrides in CSS\n    - JavaScript interactions (data-bs-* attributes)\n  RETURN usage analysis with migration complexity score\n  \n---\n\nTOOL: find_custom_bootstrap_overrides\n  \n  GLOB app/assets/stylesheets/**/*.{css,scss}\n  GLOB app/javascript/**/*.{css,scss}\n  PARSE stylesheets\n  FIND selectors targeting Bootstrap classes\n  RETURN [{file, selector, properties, specificity}]\n```\n\n### 7.
